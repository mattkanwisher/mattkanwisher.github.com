Handling blocking IO in GO.

Jul 19th, 2012

So I was trying to write a simple system monitoring server in Go and one of the problems I ran into was blocking IO, so for example if you need to spawn a sub process and listen to it’s output you need to spawn a seperate go routines to monitor it. Thats fine, however the problem is if you ever have a piece of blocking IO that never returns like say your spawning “tail -f” you will have no way of killing the goroutine. I love blocking IO, but not having a mechanism to timeout on blocking IO seems like a big deficiency. We have one channel to get data from the go routine, and another that tells the goroutine to die. Why we need this? Cause there is no way to kill a goroutine safely.
